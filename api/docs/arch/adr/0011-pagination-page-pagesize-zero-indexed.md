# Pagination uses page + pageSize, zero-indexed (page=0 is first page)

All paginated endpoints accept `page` (zero-indexed integer) and `pageSize` query parameters and return `{ data, page, pageSize, total }`. `page=0` is the first page. This matches the dominant convention in the existing Nuxt codebase (`/api/leaderboard`, `/api/.../vulnerabilities`) — porting endpoints without translation keeps the `nuxt-to-api` conversion trivial and avoids off-by-one bugs during migration. Endpoints that currently use `limit`/`offset` (e.g. `organization-leaderboard`) get normalized to `page`/`pageSize` at port time. The trade-off is that zero-indexed pagination is uncommon in public HTTP APIs (most use 1-based), which may mildly surprise external developers — this is called out prominently in the docs. `total` comes from Tinybird's `rows_before_limit_at_least` field. Changing default `pageSize` values is a breaking change per ADR-0003.
